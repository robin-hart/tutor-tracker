TutorTimeTracker API

Comprehensive REST API for tutoring session management, student tracking, and performance reporting.

Overview

TutorTimeTracker is a Spring Boot-based backend API that manages the complete lifecycle of tutoring projects, from project creation through student management, session scheduling, and monthly report generation. The API follows REST principles and provides JSON-based request/response payloads with input validation.

Architecture

Three-Tier Architecture

  • Controller Layer: REST endpoints in com.tutortimetracker.api.controller
  • Service Layer: Business logic in com.tutortimetracker.api.service
  • Persistence Layer: JPA repositories in com.tutortimetracker.api.repository
  • Model Layer: Entities in com.tutortimetracker.api.entity and DTOs in com.tutortimetracker.api.model

Database

Production: MariaDB 10.x+ (default port 3306; Docker service internal only)
Testing: H2 in-memory with MariaDB compatibility mode

Core Entities

Core Entities and Attributes
Entity Purpose Key Attributes
ProjectEntity Tutoring project (dashboard card) slug, name, category, totalHours, monthHours, completionPercent
StudentEntity Individual student in a project studentKey, name, notes, groupName, project
ProjectGroupEntity Student grouping within a project name, project (e.g., "Honors", "Group A")
TimeslotEntity Individual tutoring session id, title, date, startTime, durationMinutes, description, project
ReportRowEntity Monthly aggregated metrics monthKey, totalHours, sessions, grossAmount, project
TodaySlotEntity Today's scheduled sessions (view optimization) Similar to TimeslotEntity, project-scoped

API Endpoints

Project Management

    GET    /api/projects              - List all projects
    POST   /api/projects              - Create new project
    DELETE /api/projects/{projectId}  - Delete project (cascades to students, timeslots, reports)
    GET    /api/projects/{projectId}/calendar - Get project calendar with today's slots

Student Management

    GET    /api/projects/{projectId}/students              - List students in project
    POST   /api/projects/{projectId}/students              - Add student to project
    PATCH  /api/students/{studentId}/notes                 - Update student notes
    PATCH  /api/students/{studentId}/group                 - Change student group
    DELETE /api/students/{studentId}                        - Remove student

Group Management

    GET    /api/projects/{projectId}/groups                - List groups with student counts
    POST   /api/projects/{projectId}/groups                - Create new group
    DELETE /api/projects/{projectId}/groups/{groupName}    - Delete group (reassign students)

Timeslot Management

    GET    /api/projects/{projectId}/timeslots             - List timeslots (with optional month filter)
    GET    /api/projects/{projectId}/timeslots/{slotId}    - Get single timeslot
    POST   /api/projects/{projectId}/timeslots              - Create timeslot in project
    PUT    /api/projects/{projectId}/timeslots/{slotId}    - Update timeslot
    DELETE /api/projects/{projectId}/timeslots/{slotId}    - Delete timeslot
    POST   /api/timeslots                                   - Create global timeslot (legacy)

Report Management

    GET    /api/reports                                     - List all reports
    GET    /api/projects/{projectId}/reports                - List project reports
    POST   /api/projects/{projectId}/reports/generate       - Generate/refresh monthly report

Business Rules & Constraints

Project Rules

  • Slug is auto-generated from project name (lowercase, hyphenated)
  • Slug uniqueness is enforced; conflicts are resolved by appending "-2", "-3", etc.
  • Every project automatically gets an "Ungrouped" group on creation
  • Deleting a project cascades to all students, timeslots, groups, and reports

Student Rules

  • Notes are trimmed of leading/trailing whitespace
  • Students default to "Ungrouped" group if not specified
  • Each student must belong to exactly one group at a time
  • Student names default to empty string if not provided

Group Rules

  • The "Ungrouped" group cannot be deleted
  • When a group is deleted, all assigned students are reassigned to "Ungrouped"
  • Group names are case-insensitive normalized

Timeslot Rules

  • Duration is specified in minutes
  • Deleting a timeslot triggers project metrics refresh
  • Project hour metrics are calculated as: sum(timeslot.durationMinutes) / 60.0

Report Rules

  • Reports are generated monthly from timeslot data
  • Hourly rate is fixed at $60/hour (gross amount = hours × 60)
  • Month key format: "yyyy-MM" (e.g., "2026-01")

Error Handling

The API returns standard HTTP status codes:

  • 200 OK - Successful GET or PATCH
  • 201 Created - Successful POST
  • 204 No Content - Successful DELETE
  • 400 Bad Request - Validation error (blank notes, invalid group name, etc.)
  • 404 Not Found - Resource not found (project, student, timeslot, etc.)
  • 500 Internal Server Error - Unexpected server error

Error responses include an ApiErrorResponse with message details.

Validation

  • Student notes: Non-blank (after trimming), max 1000 characters
  • Project name: Required, trimmed
  • Timeslot duration: Positive integer (minutes)
  • All string inputs are trimmed to remove accidental whitespace

Testing

Comprehensive test coverage via Maven with two test plugins:

  • Unit Tests (*Test.java): Rapid business logic validation with mocked repositories
  • Integration Tests (*IT.java): Real H2 database with Spring Boot context

Run tests with: mvn verify

Getting Started (Development)

    1. Start MariaDB container: docker-compose -f docker-compose.yml up -d
    2. Build and run: mvn spring-boot:run
    3. API base URL: http://localhost:8080/api
    4. Frontend access: http://localhost:5173 (Vue dev server)
